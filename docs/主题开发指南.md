# 主题开发指南

PyKYCH 主题系统允许你通过覆盖 CSS 变量和模板来完全改变网站的外观，而无需修改核心代码。每个主题是一个独立的目录，包含配置文件和样式资源。

## 目录结构

```
data/themes/
└── 你的主题名/
    ├── theme.yaml          # 主题配置（必需）
    ├── static/
    │   └── theme.css       # 主题样式（必需，通过 CSS 变量定义配色）
    │   └── bg.png          # 额外静态资源（可选）
    └── templates/          # 模板覆盖（可选）
        └── home.html       # 覆盖首页模板
        └── base.html       # 覆盖基础模板
```

## 快速开始：创建一个新主题

以创建一个叫 `rose` 的主题为例：

### 1. 创建目录

```
data/themes/rose/
├── theme.yaml
└── static/
    └── theme.css
```

### 2. 编写 theme.yaml

```yaml
name: Rose
version: "1.0"
author: 你的名字
description: 玫瑰主题 — 柔和的粉红色调
```

字段说明：

| 字段 | 必需 | 说明 |
|------|------|------|
| `name` | 是 | 主题显示名称 |
| `version` | 否 | 版本号 |
| `author` | 否 | 作者署名 |
| `description` | 否 | 简短描述，会在后台主题选择器中显示 |

### 3. 编写 theme.css

主题 CSS 通过 **CSS 自定义属性（变量）** 来覆盖网站配色。你只需定义 `:root`（浅色模式）和 `[data-theme="dark"]`（深色模式）两组变量。

```css
/* Rose 玫瑰主题 */

:root {
    --bg: #fef6f8;
    --bg-card: #ffffff;
    --text: #2d1520;
    --text-secondary: #664455;
    --text-muted: #997788;
    --border: #f0dde4;
    --accent: #e85382;
    --accent-hover: #d43d6e;
    --code-bg: #fdf2f5;
    --shadow: 0 1px 3px rgba(232, 83, 130, 0.06), 0 1px 2px rgba(232, 83, 130, 0.04);
    --shadow-md: 0 4px 6px rgba(232, 83, 130, 0.05), 0 2px 4px rgba(232, 83, 130, 0.04);
    --radius: 8px;
    --header-bg: rgba(254, 246, 248, 0.94);
}

[data-theme="dark"] {
    --bg: #1a0a10;
    --bg-card: #24141a;
    --text: #f0d0d8;
    --text-secondary: #997788;
    --text-muted: #664455;
    --border: #2e1a24;
    --accent: #f4729e;
    --accent-hover: #f9a8c8;
    --code-bg: #1c0c12;
    --shadow: 0 1px 3px rgba(0, 0, 0, 0.5), 0 1px 2px rgba(0, 0, 0, 0.4);
    --shadow-md: 0 4px 6px rgba(0, 0, 0, 0.6), 0 2px 4px rgba(0, 0, 0, 0.5);
    --header-bg: rgba(26, 10, 16, 0.94);
}
```

## CSS 变量参考

主题 CSS 在 base.html 内联样式**之后**加载，因此你定义的变量会覆盖默认值。所有变量均为可选——未定义的变量将保持默认值。

### 前端变量（base.html）

| 变量 | 默认值 (浅色) | 默认值 (深色) | 用途 |
|------|-------------|-------------|------|
| `--bg` | `#ffffff` | `#000000` | 页面背景色 |
| `--bg-card` | `#ffffff` | `#111111` | 卡片背景色 |
| `--text` | `#1a1a1a` | `#e0e0e0` | 正文文字颜色 |
| `--text-secondary` | `#555555` | `#999999` | 次要文字颜色 |
| `--text-muted` | `#999999` | `#666666` | 弱化文字颜色 |
| `--border` | `#e5e5e5` | `#2a2a2a` | 边框颜色 |
| `--accent` | `#3b82f6` | `#60a5fa` | 强调色（链接、按钮等） |
| `--accent-hover` | `#2563eb` | `#93bcf8` | 强调色悬停态 |
| `--code-bg` | `#f5f5f5` | `#1a1a1a` | 代码块背景色 |
| `--shadow` | `0 1px 3px rgba(...)` | `0 1px 3px rgba(...)` | 轻微阴影 |
| `--shadow-md` | `0 4px 6px rgba(...)` | `0 4px 6px rgba(...)` | 中等阴影 |
| `--radius` | `8px` | `8px` | 圆角半径 |
| `--max-width` | `720px` | `720px` | 内容区最大宽度 |
| `--header-bg` | `rgba(255,255,255,0.92)` | `rgba(0,0,0,0.92)` | 顶部导航栏背景 |

### 后台变量（admin_base.html）

后台管理面板有独立的变量体系，以 `--wp-` 为前缀：

| 变量 | 默认值 | 用途 |
|------|--------|------|
| `--wp-bg` | `#f0f0f1` | 后台页面背景 |
| `--wp-card-bg` | `#ffffff` | 卡片背景 |
| `--wp-text` | `#1d2327` | 正文颜色 |
| `--wp-text-secondary` | `#50575e` | 次要文字 |
| `--wp-text-muted` | `#787c82` | 弱化文字 |
| `--wp-border` | `#c3c4c7` | 边框 |
| `--wp-border-light` | `#dcdcde` | 浅边框 |
| `--wp-primary` | `#2271b1` | 主色 |
| `--wp-primary-hover` | `#135e96` | 主色悬停 |
| `--wp-danger` | `#d63638` | 危险/删除色 |
| `--wp-danger-hover` | `#b32d2e` | 危险色悬停 |
| `--wp-success` | `#008a20` | 成功色 |
| `--wp-warning` | `#dba617` | 警告色 |
| `--wp-sidebar-bg` | `#1d2327` | 侧边栏背景 |
| `--wp-sidebar-hover` | `#2c3338` | 侧边栏悬停 |
| `--wp-sidebar-active` | `#2271b1` | 侧边栏激活项 |
| `--wp-sidebar-text` | `#c3c4c7` | 侧边栏文字 |
| `--wp-sidebar-icon` | `#8c8f94` | 侧边栏图标 |
| `--wp-sidebar-width` | `200px` | 侧边栏宽度 |

### 主题中覆盖后台变量

如果你希望主题同时影响后台外观，可以在 `theme.css` 中同时定义 `--wp-*` 变量：

```css
:root {
    --wp-primary: #e85382;
    --wp-primary-hover: #d43d6e;
    --wp-sidebar-bg: #2d1520;
    --wp-sidebar-active: #e85382;
    /* ... 其他后台变量 ... */
}

[data-theme="dark"] {
    --wp-bg: #1a0a10;
    --wp-card-bg: #24141a;
    --wp-text: #f0d0d8;
    /* ... 其他后台变量 ... */
}
```

## 设计配色建议

### 浅色模式设计原则

- `--bg` 使用略微偏离纯白的柔和色调（如 `#f6f8f5`），减少眼部疲劳
- `--bg-card` 保持纯白 `#ffffff`，与背景形成微妙层次
- `--accent` 饱和度适中，作为页面唯一的彩色点缀
- `--text` 不要用纯黑 `#000`，用深灰（如 `#1a1a1a`）
- `--border` 保持低对比度，用浅灰

### 深色模式设计原则

- `--bg` 用非常暗的色调（如 `#0d140c`），但不是纯黑
- `--bg-card` 比背景稍亮（如 `#151c14`），形成层次
- `--accent` 比浅色模式更亮，补偿深色背景上的对比度损失
- `--text` 用浅灰（如 `#dce5d8`），不要用纯白
- `--shadow` 加深阴影以在深色背景上可见

### 快速颜色生成技巧

1. 选一个主色调作为 `--accent`
2. 浅色模式：背景 = 主色调亮度 95% + 饱和度 10%；文字 = 主色调亮度 10%
3. 深色模式：背景 = 主色调亮度 5%；文字 = 主色调亮度 90%

## 模板覆盖

主题可以覆盖任何 Jinja2 模板文件。只需在 `templates/` 目录下放置与原始模板同名的文件。

### 可覆盖的模板

```
src/pykych/templates/
├── base.html              # 基础布局（导航栏、页脚）
├── home.html              # 首页
├── login.html             # 登录页
├── page.html              # 通用页面
├── search.html            # 搜索结果页
├── labels.html            # 标签页
├── label_detail.html      # 标签详情页
├── md_list.html           # Markdown 文章列表
├── md_detail.html         # Markdown 文章详情
├── wikidot_list.html      # Wiki 页面列表
├── wikidot_detail.html    # Wiki 页面详情
├── bbcode_list.html       # BBCode 文章列表
├── bbcode_detail.html     # BBCode 文章详情
├── html_list.html         # HTML 页面列表
├── html_detail.html       # HTML 页面详情
├── external_html.html     # 外部 HTML 页面
├── _line_comments.html    # 行评论组件
├── _rating_section.html   # 评分区组件
├── admin_base.html        # 后台基础布局
├── admin_dashboard.html   # 后台仪表盘
├── admin_profile.html     # 后台个人资料
├── admin_settings.html    # 后台设置
└── ...                    # 其他模板
```

### 模板覆盖示例

在 `data/themes/rose/templates/home.html` 中覆盖首页：

```html
{% extends "base.html" %}

{% block extra_head %}
<style>
    .custom-hero {
        background: linear-gradient(135deg, var(--accent), #f9a8c8);
        padding: 4rem 2rem;
        text-align: center;
        border-radius: var(--radius);
        margin-bottom: 2rem;
    }
</style>
{% endblock %}

{% block content %}
<div class="custom-hero">
    <h1>欢迎来到我的网站</h1>
</div>

<!-- 保留原有的文章列表逻辑 -->
{% include "original_home_content.html" %}
{% endblock %}
```

> **提示**：模板覆盖需要一定的 Jinja2 知识。如果你只需改变配色，只用 `theme.css` 就足够了。

## 静态资源

在 `static/` 目录下可以放置任何静态文件（图片、字体、额外 CSS 等），主题的 `theme.css` 是唯一自动加载的文件。

### 引用主题静态资源

如果需要在主题 CSS 中引用 `static/` 目录下的资源，使用相对路径：

```css
:root {
    --bg-image: url('../static/bg.png');
}
```

或者在模板中通过主题路由引用（注意：当前版本需要通过 `/theme.css` 中的 `@import` 或 base64 内联来加载额外资源）。

## 激活主题

### 方法一：后台设置（推荐）

1. 登录后台 → **站点设置** → **外观**
2. 在「站点风格主题」下拉菜单中选择你的主题
3. 点击「保存更改」

### 方法二：手动设置

编辑 `data/settings.yml`：

```yaml
appearance:
  theme: auto           # auto | light | dark
  style_theme: rose     # 你的主题名称（目录名）
  primary_color: "#e85382"
```

## 内置主题参考

PyKYCH 自带 6 个主题，可作为开发参考：

| 主题 | 特点 |
|------|------|
| `default` | 默认蓝白配色，`theme.css` 为空（使用 base.html 内联默认值） |
| `forest` | 森林绿，`--radius: 10px` 圆角更大 |
| `ocean` | 海洋蓝青，冷色调 |
| `sunset` | 日落橙红，暖色调 |
| `lavender` | 薰衣草紫，`--radius: 12px` 最大圆角 |
| `mono` | 极简黑白灰，`--radius: 4px` 最小圆角 |

## 最佳实践

1. **只覆盖需要的变量**：不需要定义所有变量，未定义的自动使用默认值
2. **始终定义深色模式**：即使用户只用浅色模式，也提供 `[data-theme="dark"]` 块
3. **测试两种模式**：在浅色和深色模式下都预览一遍
4. **保持对比度**：文字与背景对比度至少 4.5:1（可用浏览器 DevTools 检查）
5. **阴影适配**：深色模式的阴影应更深（`rgba(0,0,0,0.6)` vs 浅色的 `rgba(... ,0.04)`）
6. **尊重 `--max-width`**：不建议大幅修改内容区最大宽度，保持在 `680-800px` 之间

## 调试技巧

1. **浏览器 DevTools**：打开开发者工具 → Elements → 检查 `:root` 和 `[data-theme="dark"]` 的计算样式，确认变量是否生效
2. **直接访问**：打开 `http://localhost:8000/theme.css` 查看当前激活主题的 CSS
3. **临时覆盖**：在 DevTools 的 Styles 面板中直接编辑变量值来快速预览效果

## 分享你的主题

如果你制作了好看的主题，欢迎通过 Pull Request 贡献给项目！

主题目录结构要求：

```
data/themes/你的主题名/
├── theme.yaml          # name, version, author, description
├── static/
│   └── theme.css       # 完整的 :root 和 [data-theme="dark"] CSS
```
